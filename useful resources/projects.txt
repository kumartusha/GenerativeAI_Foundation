4. #######################       RAG pipeline design       #############################
Your RAG system is not slow because of the LLM.
It is slow because your pipeline is broken.

People blame hallucinations.
They blame embeddings.
They blame the model.

But real RAG lives in the layers between them.
I created this RAG Complete Cheatsheet to show how a production RAG system actually works, end to end 👇 

RAG is not one step. It is a system:
→ Indexing (Offline)
↳ Load PDFs, docs, web pages
↳ Chunk them correctly
↳ Convert them into embeddings
↳ Store them in a vector database

→ Retrieval (Real-time)
↳ User query → embedding
↳ Search vector DB
↳ Apply hybrid search (BM25 + vectors)
↳ Filter with metadata
↳ Select Top-K

→ Re-ranking
↳ First pass retrieves fast
↳ Second pass re-orders with a reranker
↳ Best chunks rise to the top

→ Generation
↳ LLM sees only the retrieved context
↳ Prompt enforces grounding
↳ Answer is generated from facts, not memory

→ Evaluation & Observability
↳ Precision, Recall, MRR
↳ Faithfulness, Answer relevance
↳ RAGAS, TruLens, DeepEval

→ Advanced RAG
↳ Multi-Query RAG
↳ Self-RAG (self-correction)
↳ Agentic RAG
↳ GraphRAG for multi-hop reasoning

→ Production Layer
↳ Caching (Redis)
↳ Security and PII handling
↳ Cost and latency tracking
↳ Re-indexing and freshness

This cheatsheet puts all of this in one place
from chunking → embeddings → retrieval → reranking → LLM → evaluation → production 

RAG is not “documents in → answer out”.
RAG is search + reasoning + verification + systems engineering.

4. #######################       RAG Interview design       #############################
𝐙𝐞𝐫𝐨𝐝𝐡𝐚 𝗮𝘀𝗸𝗲𝗱 𝘁𝗵𝗶𝘀 𝗶𝗻 𝐋𝟐 𝐫𝐨𝐮𝐧𝐝 𝐨𝐟 𝐀𝐈 𝐄𝐧𝐠𝐢𝐧𝐞𝐞𝐫 𝐑𝐨𝐥𝐞.

In financial RAG, accuracy isn’t nice-to-have, it’s everything.

You’re building RAG for financial PDFs. Missing a single number from search can cost millions. What is the best search method?

Answer:
- Use hybrid search, it’s the baseline for high-stakes data.
- Combine BM25 (keyword) + semantic search → exact numbers + contextual meaning
- Merge results using RRF (Reciprocal Rank Fusion)
- Add a reranker (e.g., Cohere Rerank) to push the most relevant chunk to the top
- Use layout-aware chunking (tables intact) and metadata filtering (company, quarter, doc type)
- Apply multi-query expansion to improve recall
- Always cite sources for every number

In finance, retrieval is risk management—hybrid search + reranking is the minimum bar.
